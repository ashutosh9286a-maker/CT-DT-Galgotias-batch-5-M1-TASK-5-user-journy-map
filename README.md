# User Journey Map

## Overview

This repository contains the User Journey Map created as part of the L&T EduTech M1 Task 5 assignment.

The User Journey Map visually represents the complete experience of a user while interacting with a product or service. It helps identify user actions, emotions, goals, pain points, and opportunities for improvement at each stage of the journey.

## Objective

The objective of this project is to understand the user experience from awareness to post-purchase interaction and identify areas where the product or service can be improved.

## Journey Stages

* Awareness
* Consideration
* Product Exploration
* Customization
* Decision Making
* Purchase
* Product Usage
* Feedback and Support

## User Journey Components

* User Actions
* User Goals
* User Thoughts
* User Emotions
* Pain Points
* Opportunities for Improvement

## Journey Flow

1. User becomes aware of the product or service.
2. User visits the platform and explores available options.
3. User selects and customizes the product according to requirements.
4. User reviews the configuration and pricing.
5. User proceeds to purchase and completes payment.
6. User receives and uses the product.
7. User provides feedback or contacts support if required.
8. Insights are collected to improve the overall user experience.

## Benefits of User Journey Mapping

* Better understanding of user behavior
* Identification of pain points
* Improved customer satisfaction
* Enhanced product design decisions
* Better user experience optimization
